AR text conditioning, without the fog

EOSTok-style loop Hash removed Old cross-attn baseline Prefix AR implemented

Verdict: the repo now uses the more AR-native path: projected Qwen3.5 text tokens live inside the same causal sequence as image tokens.

That means the AR model does not receive a class id or a side channel. It reads prompt tokens first, then predicts image codes one by one.

Three conditioning styles

Weak baseline

Caption hash / class id

A caption becomes an arbitrary integer. The model sees a label embedding, not language.

Previous baseline

Frozen Qwen + cross-attn

Caption hidden states condition visual token prediction through cross-attention and pooled AdaLN.

Current repo

Text tokens in the sequence

The prompt is prefix context. Image codes are generated after it with one causal attention rule.

The sequence idea

One training row

Given one image-caption pair, the tokenizer gives image codes. The AR model sees the caption prefix and predicts only the visual code positions.

<bos> a brown dog grass <img> v1 v2 v3 v4
prompt prefix visual loss positions future tokens blocked 
input  = [text tokens, IMG_START, shifted image tokens]
target = [ignore text, ignore IMG_START, image codes]
loss   = cross_entropy(logits_at_image_positions, image_codes)

Why this is more AR-native

Cross-attention says: image stream, look sideways at text features.

Prefix AR says: text came before the image, so every visual token naturally attends to the prompt through causal self-attention.

That matches how language models condition generation: no special class embedding, no caption hash, no separate conditioning branch required.

Causal mask for prefix AR

The row is the token being updated. The column is what it can read. Image tokens can read all prompt tokens and earlier image tokens, but not future image tokens.

What is integrated into ImageGen now

  1. Keep the EOSTok-style image tokenizer path. The image still becomes compact 1D codes through encoder, IBQ, and decoder.
  2. Replace cross-attn with prefix self-attn. Project Qwen hidden states into AR width, prepend them to the image-token stream, and run one causal transformer.
  3. Compute loss only on image positions. Text prefix is context. The model is judged on predicted visual codes.
  4. Use empty text prefix for CFG. Conditional and unconditional batches share the same visual-token path.
  5. Later upgrade the tokenizer. ViQ, ARM, or UniAR-style visual tokens are the next research path after prefix conditioning works.

Current architecture

text = qwen(captions).hidden_states[-1]
text = text_proj(text)

img_in = concat(IMG_START, image_token_embs[:, :-1])
x = concat(text, img_in)

h = causal_transformer(x, mask=prefix_ar_mask)
visual_h = h[:, text_len:]
logits = visual_head(visual_h)
loss = CE(logits, image_code_ids)

This is the implemented middle path: keep frozen Qwen for text understanding, but make the AR transformer consume text as prefix context instead of side-channel cross-attention.

Bring in ViQ or UniAR-style text-aligned visual tokens. This attacks the biggest bottleneck after text conditioning: the image token space.
